# Bootstrap Script — Implementation Plan

## Goal

A single `install.sh` at the repo root that sets up a new machine end-to-end with no manual steps.

## Full install flow

```
dotfiles/install.sh        ← steps 1–5: personal setup + verification
dotfiles-work/install.sh   ← step 6: work overlay (run after personal)
```

---

### 1. Detect OS

```bash
case "$(uname -s)" in
  Darwin) PLATFORM="osx" ;;
  Linux)
    grep -qi microsoft /proc/version && PLATFORM="wsl2" || PLATFORM="linux"
    ;;
  *) echo "Unsupported OS"; exit 1 ;;
esac
```

### 2. Install prerequisites

Update git to latest version first, then install tooling.

**macOS:**
- Check for Xcode CLI tools (`xcode-select -p`), install if missing
- Check for Homebrew, install if missing
- `brew upgrade git`
- `brew install stow`

**Linux/WSL2:**
- `sudo apt-get update && sudo apt-get upgrade -y`
- `sudo add-apt-repository ppa:git-core/ppa && sudo apt-get update`
- `sudo apt-get install -y git stow`

### 3. Stow personal dotfiles

```bash
cd "$(dirname "$0")"
for dir in git zsh shared "$PLATFORM"; do
  stow --restow "$dir"
done
```

### 4. Install packages

Everything is managed via Homebrew on all platforms (macOS, Linux, WSL2):

```bash
brew bundle --file=osx/Brewfile
```

### 5. Verify personal setup

Run after stowing and installing. SSH checks are critical — the work repo clone in step 6 depends on them passing.

**Git config (non-critical):**
```bash
echo "==> Git identity"
git config --global user.name
git config --global user.email
```

**SSH connections (critical):**
```bash
echo "==> GitHub SSH"
ssh -T git@github.com 2>&1 | grep -q "successfully authenticated" \
  && echo "OK" || echo "FAILED — check 1Password SSH agent"

echo "==> Personal server SSH"
ssh -T git@personal-server 2>&1 | grep -q "..." \
  && echo "OK" || echo "FAILED"
```

If any SSH check fails, abort and print a message to open 1Password and enable the SSH agent before continuing.

### 6. Clone and install dotfiles-work

After step 5 confirms SSH is working, the private work repo can be cloned automatically.

```bash
DOTFILES_WORK_REPO="git@github.com:WilberC/dotfiles-work.git"
DOTFILES_WORK_DIR="$HOME/Projects/Personal/dotfiles-work"

if [ ! -d "$DOTFILES_WORK_DIR" ]; then
  git clone "$DOTFILES_WORK_REPO" "$DOTFILES_WORK_DIR"
fi

bash "$DOTFILES_WORK_DIR/install.sh"
```

`dotfiles-work/install.sh` stows every company package and runs `refresh.sh` to generate `~/.config/gitconfig/work.conf`.

### 7. Verify work setup

**Git config (non-critical):**
```bash
echo "==> Work git identity (inside a work repo)"
cd ~/Projects/Work/Outcode
git config user.name
git config user.email
```

**SSH connections (critical):**
```bash
echo "==> Azure DevOps SSH"
ssh -T git@ssh.dev.azure.com 2>&1 | grep -q "authenticated" \
  && echo "OK" || echo "FAILED — check Outcode SSH key in 1Password"
```

If any SSH check fails, print a message pointing to the relevant key in 1Password.

### 8. Post-install hints

Print a short checklist reminding the user to:
- Set up `mise` tool versions
- Sign into GitHub CLI (`gh auth login`)

## Adding a new company to dotfiles-work

Only `dotfiles-work` needs to change — personal dotfiles are never touched.

1. Create the company package folder:
   ```
   dotfiles-work/<company>/.config/gitconfig/accounts/<company>        ← [user] identity
   dotfiles-work/<company>/.config/gitconfig/accounts/<company>.gitdir ← path that triggers identity
   dotfiles-work/<company>/.config/ssh/configs/<company>-*.conf        ← SSH host config (if needed)
   dotfiles-work/<company>/.config/ssh/pubs/<company>.pub              ← public key (if needed)
   ```
2. Stow and refresh:
   ```bash
   cd ~/Projects/Personal/dotfiles-work
   stow <company>
   bash refresh.sh
   ```

SSH configs are picked up automatically by the `Include ~/.config/ssh/configs/*.conf` glob in `~/.ssh/config`. Git routing is auto-generated by `refresh.sh` scanning `*.gitdir` files.

## File layout

```
dotfiles/
  install.sh        ← personal entry point (to be written)
  osx/Brewfile      ← to be written

dotfiles-work/      ← private repo, gitignored here
  install.sh        ← work entry point (done)
  refresh.sh        ← regenerates ~/.config/gitconfig/work.conf (done)
  outcode/          ← Outcode stow package (done)
```

## Order of implementation

1. Write `dotfiles/install.sh` with OS detection and stow logic
2. Add `osx/Brewfile` and wire into `install.sh`
3. Add personal SSH + git verification step
4. Add dotfiles-work clone + install step
5. Add work SSH + git verification step
6. Add post-install hint output
